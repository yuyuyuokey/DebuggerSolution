# 参考文献阅读指南

> 原则：你不需要读完 40 篇论文。你需要的是——用最少的时间，提取对写论文最有用的信息。

---

## 一、阅读策略：三遍法

### 第一遍（每篇 10 分钟）—— 筛选：这篇对我有用吗？

只读三个东西：
1. **Abstract（摘要）**：作者做了什么？解决了什么问题？
2. **Introduction 最后一段**：作者列出了哪几个 contribution？
3. **Figures/Tables**：扫一眼所有图和表

读完这三样，你能判断这篇论文对你有没有用。大部分论文在这一轮就会被筛掉。

### 第二遍（每篇 30-60 分钟）—— 提取：我要引用它的什么？

只读三个部分：
1. **Related Work / Background**：可以抄来用在你自己的 1.2 节和 2.5 节
2. **Method / Architecture**：作者的系统怎么设计的？和你的有什么异同？
3. **Experiment / Results**：作者用什么指标？做了什么对比？你可以参考他的实验设计

### 第三遍（每篇 1-2 小时）—— 精读：只针对和你直接相关的 3-5 篇

只精读和你的工作**直接可比**的论文，比如 ChatDBG、REx86、ReCopilot。

## 三、英文论文阅读技巧

### 3.1 看不懂怎么办？

**核心原则：你不是在学英语，你是在提取信息。用工具，不要硬啃。**

1. **先用翻译工具读 Abstract**：DeepL / ChatGPT / Claude 翻译摘要，30 秒判断这篇值不值得继续读
2. **只读你能用上的段落**：大多数论文 80% 的内容（公式推导、实验细节、消融实验分析）你不需要理解
3. **重点抓"对比句"**：论文里出现 "Unlike..." / "In contrast to..." / "Different from..." 的句子，通常就是你能用来写"现有工作的不足"的素材
4. **直接看 Figure 标题**：图标题通常用一句话概括了核心发现

### 3.2 你需要从每篇论文里提取什么？

对每篇 Tier 1/2 论文，填这个卡片就够了：

```
论文：[标题]
作者/会议/年份：[...]
一句话总结：[作者做了什么，用了什么方法，达到了什么效果]
与我的关系：[我的论文为什么需要引用它]
可引用的具体句子：[从 Abstract/Conclusion 里复制 1-2 句英文原文]
```

### 3.3 学术英语的常见套路

| 你看到的英文 | 中文意思 | 出现在哪 |
|-------------|---------|---------|
| "We propose a novel framework..." | 作者提了一个新框架 | Abstract/Introduction |
| "Unlike prior work that..." | 和之前的工作不同之处 | Introduction/Related Work |
| "Our approach achieves X% improvement over..." | 比基线方法提升了多少 | Abstract/Experiment |
| "However, existing methods suffer from..." | 现有方法的不足 | Introduction |
| "We leave ... for future work" | 作者承认的局限性 | Conclusion |

## 四、我帮你读——最核心 4 篇的中文要点

### [论文 1] ChatDBG (FSE 2025) —— 最直接相关

**作者做了什么**：把 GPT-4 集成到 GDB/LLDB/Pdb（Python 调试器）中。用户问"为什么 x 是 null？"，AI 自动执行调试器命令（backtrace、print、step），分析结果，给出根因诊断。

**核心数字**：
- 67% 的 Python bug 一次对话修复
- 85% 一次追问修复
- 已获 85,000+ 下载
- 获 ACM FSE 2025 Distinguished Artifact Award

**和你的本质区别**（这是你论文 1.2 节的核心论点）：
1. ChatDBG 调试的是**有源码、有调试符号**的程序（Python/C/C++），崩溃时有变量名、有行号、有堆栈符号
2. 你调试的是**无源码、无符号、只有汇编和内存**的二进制程序
3. ChatDBG 的 LLM 看到的是"变量 x 在第 42 行为 null"，你的 LLM 看到的是"EAX=0x00000000, EIP=0x00401234, 前面的指令是 CMP EAX, EBX; JNE 0x00401300"
4. 二进制调试的语义推断难度远高于源码调试

**你该引用的原句**（可从 Abstract 提取）：
> "ChatDBG integrates LLMs into standard debuggers as an autonomous agent that can query and control the debugger to perform root cause analysis."

---

### [论文 2] REx86 (ACSAC 2025) —— 场景最接近

**作者做了什么**：用 ~6,000 个 x86 汇编样本微调 Qwen2.5-Coder-7B（开源模型），得到一个叫 REx86 的模型。输入 x86 汇编代码，输出功能解释、算法识别、用途推测。

**核心数字**：
- 交叉熵损失降低 64.2%
- 语义相似度提升 20.3%
- 用户实验（43 人）：正确率从 31% 提升到 53%（p=0.031，有统计学意义）
- 完全本地运行，不需要联网

**和你的本质区别**：
1. REx86 是**本地微调模型**做汇编解释，你是**云端模型通过 Function Calling** 主动调用调试工具
2. REx86 是**单次输入→单次输出**（给汇编→得解释），你是**多轮交互闭环**（AI 选择工具→获得新信息→继续推理）
3. REx86 是**被动解释**（你问它答），你是**主动分析**（AI 决定要读什么、什么时候读）
4. REx86 面向的是**理解汇编片段**，你面向的是**完整的逆向分析流程**

**你该引用的点**：
- 证明了 LLM 在 x86 汇编理解上确实有效（给你论文章节 2.5 提供依据）
- 用户实验的设计方法（给你论文章节 5.3 提供参考）

---

### [论文 3] Wang et al. Survey —— 给你 1.2 节搭框架

**作者做了什么**：一篇综述，把现有的"LLM + 程序分析"论文全部梳理了一遍，按以下维度分类：
- 静态分析 vs 动态分析 vs 混合方法
- 源码级 vs 二进制级
- 辅助工具型 vs 端到端生成型

**对你最有用的部分**：
- Section 2（分类法）：你可以直接借鉴这个分类框架来组织你 1.2 节的"国内外研究现状"
- Section 4（LLM for Dynamic Analysis）：这里是和你工作最相关的部分，你会发现**目前 LLM + 动态调试的论文几乎没有**——这是你论文的切入点

**你该怎么用**：
- 在 1.2 节开头引用这篇综述："根据 Wang et al. (2025) 的分类，LLM 辅助程序分析可分为..."
- 在综述末尾指出空白："但该综述也指出，LLM 在运行时调试器中的实时交互应用尚属研究空白"

---

### [论文 4] ReCopilot (2025) —— 二进制分析专业模型

**作者做了什么**：奇安信技术研究院训练了一个 7B 参数的专家模型，专门做二进制分析。三阶段训练（CPT→SFT→DPO），支持 5 个任务：
1. 反编译（汇编→C 代码）
2. 函数名恢复
3. 变量名和类型恢复
4. 结构体恢复
5. 二进制代码摘要

**核心数字**：在综合基准上比现有领域模型高 13%，从 DeepSeek-V3 (671B) 到它 (7B) 的性能下降很小。

**和你的关系**：你的工作可以和 ReCopilot 形成**互补**——ReCopilot 做的是静态分析（看汇编生成解释），你做的是动态分析（AI 操作调试器实时交互）。在论文 6.2 节展望中可以提：未来可以尝试用 ReCopilot 这样的本地模型替代 DeepSeek API 来解决隐私问题。

## 七、重要提醒

1. **不要从头到尾读论文**。学术论文的结构就是为了让你跳着读：Abstract → Introduction 最后一段 → 扫图 → 翻 Conclusion → 读 Method 开头
2. **翻译工具是你的朋友**。用 DeepL/ChatGPT/Kimi 翻译关键段落，不要有心理负担
3. **你不需要理解所有技术细节**。大部分论文的公式/算法细节你不需要懂，你只需要知道"他们做了什么"和"和你的工作有什么关系"
4. **先写 1.2 节，再啃论文**。写完初稿，知道哪里缺引用，再去针对性读那篇论文
